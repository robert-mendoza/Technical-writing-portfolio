# Technical Documentation Portfolio

A professional technical documentation portfolio showcasing user-focused documentation created using Markdown, Material for MKDocs, Git, and GitHub Pages.

This portfolio demonstrates practical technical writing skills through documentation projects that emphasize clarity, consistency, accessibility, and maintainability.

## Portfolio Overview

The portfolio includes examples of:

- User Guides
- Standard Operating Procedures (SOPs)
- Process Documentation
- Technical Specifications
- Documentation Standards
- Documentation Workflow

The documentation follows structured writing practices based on the Microsoft Writing Style Guide and the Google Developer Documentation Style Guide.

---

## Documentation Objectives

This portfolio demonstrates the ability to:

- Create task-oriented documentation.
- Transform complex technical information into clear and understandable content.
- Apply consistent document structure and formatting.
- Produce accessible documentation for screen reader users.
- Manage documentation using Git and GitHub.
- Publish documentation with Material for MKDocs.

## Accessibility

Accessibility is an important part of this portfolio.

Documentation is designed to support:

- Screen reader users
- Keyboard-only navigation
- Logical heading hierarchy
- Descriptive hyperlinks
- Accessible tables
- Plain language principles

Accessibility testing is performed using the NVDA screen reader.
