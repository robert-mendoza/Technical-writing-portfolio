    User Guide: Using NVDA with Google Docs¶

    Document Information¶

    Field Value
    Document Title Using NVDA with Google Docs
    Document Type User Guide
    Version 1.0
    Author Robert Mendoza
    Date June 2026
    Intended Audience Blind and visually impaired users, accessibility testers, educators, technical writers

    Introduction¶

    Google Docs is a cloud-based word processing application that allows users to create, edit, review, and collaborate on documents online.

    NVDA (NonVisual Desktop Access) is a free and open-source screen reader that enables blind and visually impaired users to interact with digital content using speech and keyboard commands.

    This guide explains how to use NVDA effectively with Google Docs to create accessible documents and improve productivity.

    Purpose¶

    The purpose of this guide is to provide detailed instructions for using Google Docs with NVDA.

    This guide aims to:

    • Improve navigation efficiency.

    • Support accessible document creation.

    • Reduce usability barriers.

    • Promote accessibility best practices.

    Scope¶

    This guide covers:

    • Accessing Google Docs

    • Document creation

    • Editing content

    • Formatting text

    • Accessibility techniques

    • Collaboration workflows

    • Troubleshooting common issues

    This guide does not cover:

    • Advanced scripting

    • Google Workspace administration

    • Third-party extensions

    Intended Audience¶

    This guide is intended for:

    • Blind users

    • Low vision users

    • Students

    • Educators

    • Accessibility testers

    • Technical writers

    • Documentation specialists

    Prerequisites¶

    Before using this guide, ensure the following requirements are met:

    • NVDA is installed and operational.

    • A Google account is available.

    • Internet connectivity is available.

    • Basic keyboard skills are understood.

    System Requirements¶

    Hardware Requirements¶

    • Computer with keyboard

    • Headphones or speakers

    Software Requirements¶

    • Windows 10 or later

    • NVDA latest stable version

    • Google Chrome (recommended)

    • Microsoft Edge (supported)

    Getting Started¶

    Opening Google Docs¶

    1. Open Google Chrome.

    2. Press Ctrl+L.

    3. Type docs.google.com.

    4. Press Enter.

    5. Sign in to your Google account if prompted.

    Creating a New Document¶

    1. Navigate to the Google Docs home page.

    2. Press Tab until a "Blank" document option is announced. Note: the exact NVDA announcement may vary (for example, "Start a new document, Blank"). Listen for the word "Blank" to identify the correct option.

    3. Press Enter.

    4. Wait for the editor to load.

    Expected Result:

    A new untitled document opens.

    Navigating Google Docs with NVDA¶

    Browse Mode and Focus Mode¶

    Google Docs functions best when NVDA is in Focus Mode. Focus Mode allows keyboard input to pass directly to the application.

    Toggle Focus/Browse Mode: NVDA+Space

    NVDA Speech Commands¶

    Use the following NVDA commands to control speech within the document:

    Say All (continuous reading): NVDA+DownArrow

    Read current line: NVDA+UpArrow

    Stop speech: Ctrl

    Read current word: NVDA+Ctrl+NumpadAdd

    Reading Text — Caret Navigation¶

    Character¶

    Previous character: Left Arrow

    Next character: Right Arrow

    Word¶

    Previous word: Ctrl+Left Arrow

    Next word: Ctrl+Right Arrow

    Line¶

    Previous line: Up Arrow

    Next line: Down Arrow

    Paragraph¶

    Previous paragraph: Ctrl+Up Arrow

    Next paragraph: Ctrl+Down Arrow

    Moving Through the Document¶

    Beginning of line: Home

    End of line: End

    Beginning of document: Ctrl+Home

    End of document: Ctrl+End

    Creating and Editing Documents¶

    Entering Text¶

    1. Place focus in the document.

    2. Begin typing.

    NVDA announces text according to configured speech settings.

    Selecting Text¶

    Character selection¶

    Select character: Shift+Arrow Keys

    Word selection¶

    Select word: Ctrl+Shift+Arrow Keys

    Select all¶

    Select all: Ctrl+A

    Copy, Cut, and Paste¶

    Copy: Ctrl+C

    Cut: Ctrl+X

    Paste: Ctrl+V

    Undo: Ctrl+Z

    Redo: Ctrl+Y

    Formatting Documents¶

    Applying Bold¶

    Bold: Ctrl+B

    Applying Italics¶

    Italics: Ctrl+I

    Applying Underline¶

    Underline: Ctrl+U

    Applying Headings¶

    Heading 1: Ctrl+Alt+1

    Heading 2: Ctrl+Alt+2

    Heading 3: Ctrl+Alt+3

    Heading 4: Ctrl+Alt+4

    Heading 5: Ctrl+Alt+5

    Heading 6: Ctrl+Alt+6

    Why Headings Matter¶

    Headings:

    • Improve navigation.

    • Improve accessibility.

    • Support screen reader users.

    • Create document structure.

    Working with Lists¶

    Creating Bulleted Lists¶

    1. Place cursor on a new line.

    2. Press Ctrl+Shift+8.

    Creating Numbered Lists¶

    1. Place cursor on a new line.

    2. Press Ctrl+Shift+7.

    Best Practice¶

    Use built-in list functions instead of manually typing symbols.

    Working with Tables¶

    Inserting a Table¶

    1. Open the Insert menu using the keyboard: press Alt+/ to open the command search and type "Table", or press Alt+Shift+F to access the menu bar and navigate to Insert.

    2. Select Table.

    3. Choose required dimensions.

    Navigating Tables¶

    Next cell: Tab

    Previous cell: Shift+Tab

    Accessibility Recommendations¶

    • Use header rows.

    • Keep tables simple.

    • Avoid merged cells.

    Working with Links¶

    Inserting Hyperlinks¶

    1. Select text.

    2. Press Ctrl+K.

    3. Enter URL.

    4. Press Enter.

    Link Text Best Practices¶

    Use descriptive link text that clearly identifies the destination or action:

    Correct: Download Accessibility Guide

    Avoid: Click Here

    Also ensure each link's text is unique within the document. Multiple links with identical text (for example, two links both labelled "Read More") cannot be distinguished by NVDA users navigating by links. Each link should have unique, descriptive text.

    Adding Images and Alternative Text¶

    Insert Image¶

    1. Open Insert menu.

    2. Select Image.

    3. Choose image source.

    Add Alternative Text¶

    1. Select image.

    2. Open image options.

    3. Add descriptive alternative text.

    Alternative Text Examples¶

    Good: "NVDA screen reader settings dialog."

    Poor: "Image."

    Reviewing Documents¶

    Spell Check¶

    1. Open Tools menu.

    2. Select Spelling and Grammar.

    Reviewing Headings¶

    Verify:

    • Heading hierarchy is logical.

    • No skipped heading levels.

    • Sections are properly organized.

    Collaboration Features¶

    Sharing Documents¶

    1. Open the Share dialog using the keyboard: press Ctrl+Shift+A, or Tab to the Share button and press Enter.

    2. Enter email address.

    3. Select permissions.

    4. Send invitation.

    Commenting¶

    1. Select text.

    2. Press Ctrl+Alt+M.

    3. Enter comment.

    4. Submit comment.

    Accessibility Best Practices¶

    Always:

    • Use heading styles.

    • Use descriptive and unique links.

    • Add alternative text.

    • Use proper lists.

    • Maintain logical reading order.

    • Test with NVDA before publishing.

    Avoid:

    • Excessive formatting.

    • Color-only instructions.

    • Empty headings.

    • Complex tables.

    • Generic or duplicate link text.

    Troubleshooting¶

    Issue: NVDA Does Not Read Document Content¶

    Possible Causes:

    • Focus Mode is not active

    • Browser issue

    • Page loading problem

    Resolution:

    1. Press NVDA+Space to switch to Focus Mode. Focus Mode is required for NVDA to interact with the Google Docs editor.

    2. Refresh the page.

    3. Restart the browser.

    Issue: Keyboard Commands Not Working¶

    Resolution:

    1. Verify keyboard focus is inside the document area.

    2. Verify NVDA is running.

    3. Test in another document.

    Issue: Slow Performance¶

    Resolution:

    1. Close unnecessary browser tabs.

    2. Restart NVDA.

    3. Clear browser cache.

    Frequently Asked Questions¶

    Q: Which browser works best with NVDA and Google Docs?

    A: Google Chrome is generally recommended for the best compatibility with NVDA and Google Docs.

    Q: Does Google Docs automatically save documents?

    A: Yes. Changes are automatically saved to Google Drive. You can also press Ctrl+S to force an immediate save.

    Q: Can NVDA read comments?

    A: Yes. Comments can be accessed and reviewed using Google Docs collaboration features. Press Ctrl+Alt+M to add a comment and use the commenting panel to navigate existing comments.

    Q: What do I do if NVDA reads too much or too little?

    A: Adjust NVDA's verbosity settings. Press NVDA+Ctrl+P to open Preferences, then navigate to Speech Settings. You can also cycle punctuation levels with NVDA+P.

    Q: Does NVDA work with all Google Docs features?

    A: Most core features are accessible. Some advanced features such as drawing tools and certain sidebar panels may have limited accessibility. Use the keyboard-based workflows described in this guide for the best experience.

    Q: Can I use Google Docs with NVDA on a mobile device?

    A: This guide covers Windows desktop usage with NVDA. Mobile screen readers (TalkBack on Android, VoiceOver on iOS) require different workflows and are outside the scope of this guide.

    Appendix¶

    Accessibility Validation Checklist¶

    Use this checklist before publishing or sharing any Google Docs document created with NVDA:

    □ Heading structure verified

    □ Lists properly formatted

    □ Tables accessible

    □ Alternative text added to all images

    □ Links descriptive and unique

    □ Links verified (each link tested and confirmed working)

    □ Reading order verified

    □ Spell check completed

    □ Keyboard shortcuts tested

    □ NVDA review completed

    □ Final accessibility validation completed

    Conclusion¶

    Following the procedures in this guide helps users create accessible, structured, and maintainable documents using NVDA and Google Docs. Proper use of headings, lists, tables, alternative text, and accessibility validation improves usability for all users and supports compliance with accessibility best practices.
