## Introduction

Google Docs is a cloud-based word processing application that allows
users to create, edit, review, and collaborate on documents online.

NVDA (NonVisual Desktop Access) is a free and open-source screen reader
that enables blind and visually impaired users to interact with digital
content using speech and keyboard commands.

This guide explains how to use NVDA effectively with Google Docs to
create accessible documents and improve productivity.

The purpose of this guide is to provide detailed instructions for using

## Getting Started

## Opening Google Docs

1.  Open Google Chrome.

2.  Press Ctrl+L.

3.  Type docs.google.com.

4.  Press Enter.

5.  Sign in to your Google account if prompted.

## Creating a New Document

6.  Navigate to the Google Docs home page.

7.  Press Tab until a "Blank" document option is announced. Note: the
    exact NVDA announcement may vary (for example, "Start a new
    document, Blank"). Listen for the word "Blank" to identify the
    correct option.

8.  Press Enter.

9.  Wait for the editor to load.

**Expected Result:**

*A new untitled document opens.*

## Navigating Google Docs with NVDA

## Browse Mode and Focus Mode

Google Docs functions best when NVDA is in Focus Mode. Focus Mode allows
keyboard input to pass directly to the application.

> Toggle Focus/Browse Mode: **NVDA+Space**

## NVDA Speech Commands

Use the following NVDA commands to control speech within the document:

> Say All (continuous reading): **NVDA+DownArrow**
>
> Read current line: **NVDA+UpArrow**
>
> Stop speech: **Ctrl**
>
> Read current word: **NVDA+Ctrl+NumpadAdd**

## Link Text Best Practices

Use descriptive link text that clearly identifies the destination or
action:

Correct: Download Accessibility Guide

Avoid: Click Here

Also ensure each link's text is unique within the document. Multiple
links with identical text (for example, two links both labelled "Read
More") cannot be distinguished by NVDA users navigating by links. Each
link should have unique, descriptive text.

## Accessibility Best Practices

Always:

- Use heading styles.

- Use descriptive and unique links.

- Add alternative text.

- Use proper lists.

- Maintain logical reading order.

## Troubleshooting

## Issue: NVDA Does Not Read Document Content

Possible Causes:

- Focus Mode is not active

- Browser issue

- Page loading problem

Resolution:

39. Press NVDA+Space to switch to Focus Mode. Focus Mode is required for
    NVDA to interact with the Google Docs editor.

40. Refresh the page.

41. Restart the browser.

## Issue: Keyboard Commands Not Working

Resolution:

42. Verify keyboard focus is inside the document area.

43. Verify NVDA is running.

44. Test in another document.

## Issue: Slow Performance

Resolution:

45. Close unnecessary browser tabs.

46. Restart NVDA.

47. Clear browser cache.

## Frequently Asked Questions

**Q: Which browser works best with NVDA and Google Docs?**

A: Google Chrome is generally recommended for the best compatibility
with NVDA and Google Docs.

**Q: Does Google Docs automatically save documents?**

A: Yes. Changes are automatically saved to Google Drive. You can also
press Ctrl+S to force an immediate save.

**Q: Can NVDA read comments?**

A: Yes. Comments can be accessed and reviewed using Google Docs
collaboration features. Press Ctrl+Alt+M to add a comment and use the
commenting panel to navigate existing comments.

**Q: What do I do if NVDA reads too much or too little?**

A: Adjust NVDA's verbosity settings. Press NVDA+Ctrl+P to open
Preferences, then navigate to Speech Settings. You can also cycle
punctuation levels with NVDA+P.

**Q: Does NVDA work with all Google Docs features?**

A: Most core features are accessible. Some advanced features such as
drawing tools and certain sidebar panels may have limited accessibility.
Use the keyboard-based workflows described in this guide for the best
experience.

**Q: Can I use Google Docs with NVDA on a mobile device?**

A: This guide covers Windows desktop usage with NVDA. Mobile screen
readers (TalkBack on Android, VoiceOver on iOS) require different
workflows and are outside the scope of this guide.
